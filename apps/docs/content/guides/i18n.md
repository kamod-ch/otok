# Internationalization

Otok does not ship a core i18n runtime. Use optional route segments and loaders.

## Locale in the URL

```text
src/app/routes/[[lang]]/about.tsx   → /about and /:lang/about
```

```tsx
const messages = {
  en: { title: "About" },
  de: { title: "Über uns" },
} as const;

export const loader = ({ params }) => {
  const lang = params.lang === "de" ? "de" : "en";
  return { lang, copy: messages[lang] };
};

export const head = ({ data }) => ({
  title: data.copy.title,
  lang: data.lang,
});

export default function About({ data }) {
  return <h1>{data.copy.title}</h1>;
}
```

## Typed links

```ts
import { route } from "virtual:otok-routes";

route("/[[lang]]/about", { params: { lang: "de" } });
```

## Recommendations

- Keep message catalogs at the app layer (JSON, TS modules, or a library).
- Prefer URL locale for shareable links; cookie locale can seed defaults.
- Do not add i18n dependencies to Otok core.
